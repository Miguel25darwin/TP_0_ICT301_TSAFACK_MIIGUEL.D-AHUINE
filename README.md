#ce que j'ai compris des principes des solides 
*SRP(Single Responsiblity Principale)
  une classe ne doit avoir qu'une seule responsabilité.
  cela permet un code plus clair,plus facile à maintenir et à modifier.
*OCP(open/closed principle)
  une classe doit etre ouverte à l'extention mais fermée à la modification.
  on ajoute de nouvelle fonctionnalité sans modifier le code existant.
*LSP(Liskov Substitution Principle)
  une classe fille doit pouvoir remplacer sa classe mère sans casser le program>
*ISP(Interface Segregation Principle)
  il vaut mieux avoir plusieurs petites interfaces spécifiques qu'une seule int>
*DIP(Dependency Inversion Principle)
  les classes doivent dependre d'abstractions et non de la classe concrètes
